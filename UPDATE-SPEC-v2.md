# Claudy v2 — Спецификация обновления «Отношения»

> Claudy помнит пользователя, дарит ему подарки и говорит вещи, которые создают ощущение связи — без тамагочи-механик, без обязательств, без шкал.

---

## 0. Главный принцип

Это обновление не добавляет механики, которые *требуют* от пользователя действий. Пользователь ничего не «должен» делать. Но если он есть — Claudy это замечает. И постепенно их сосуществование приобретает форму.

Claudy формирует привычки по отношению к пользователю, а не пользователь по отношению к Claudy.

---

## 1. Настройки: имя пользователя

В окно Settings добавляется новое поле:

| Setting | Тип | Default |
|---------|-----|---------|
| User name | текстовое поле | пустое |

Если имя задано — Claudy использует его в репликах (см. разделы 3, 4, 5). Если пустое — реплики формулируются без имени (фоллбэк-варианты для каждой фразы).

Сохраняется в `~/.claudy/settings.json` как `"user_name": "..."`.

Важно, что имя пользователя может быть набрано как кириллицей, так и латиницей.

## 2. Память

### 2.1 Файл памяти

Claudy ведёт файл `~/.claudy/memory.json`, который обновляется в реальном времени (запись при каждом значимом событии или периодически, например, раз в несколько минут).

Структура:

```json
{
  "first_launch": "2026-04-04",
  "total_days": 47,
  "today": {
    "date": "2026-04-04",
    "clicks": 3,
    "app_launches": {
      "com.vivaldi.Vivaldi": 2,
      "com.googlecode.iterm2": 5,
      "com.spotify.client": 1
    }
  },
  "gifts": [
    { "type": "fish", "emoji": "🐟", "date": "2026-04-03", "collected": true },
    { "type": "star", "emoji": "⭐", "name": "Звезда имени Kate", "date": "2026-04-04", "collected": false }
  ]
}
```

### 2.2 Подсчёт дней

`total_days` — количество уникальных дней, когда Claudy был запущен. При запуске приложения значение сбрасывается в 0 и далее обновляется (инкремент на единицу) раз в сутки.

### 2.3 Подсчёт кликов

`today.clicks` — количество **одиночных** кликов по Claudy за сегодня. Двойные клики (вызов Claude.app) и вызовы контекстного меню **не считаются**. Счётчик сбрасывается при смене дня.

**Порог привязанности: 5 кликов в день.** Этот порог невидим для пользователя. Влияние описано в разделе 3.

### 2.4 Подсчёт запусков приложений

`today.app_launches` — словарь `{ bundleID: count }`. Обновляется при каждом `didLaunchApplicationNotification`. Используется для реплик с памятью (раздел 5).

---

## 3. Привязанность (порог кликов)

Порог: **5 одиночных кликов в день**. До порога — обычное поведение. После порога — три эффекта:

### 3.1 Частицы при клике

| Клики за день | Частицы при одиночном клике |
|---------------|---------------------------|
| ≤ 5 | sparkle (звёздочки, блёстки) |
| > 5 | heart (сердечки) — как сейчас |

Переключение происходит автоматически при превышении порога. Если на следующий день кликов снова < 5, возвращаются sparkle.

### 3.2 Персональные реплики

После превышения порога в пуле фраз для реакции на клик **добавляются** (не заменяют!) персональные фразы:

```
RU:
  "как дела, {name}?"
  "чем занимаешься, {name}?"
  "мне нравится проводить время с {name} :3"
  "рада тебя видеть, {name}!"
  "{name}! :3"
  "а {name} сегодня внимательный"

EN:
  "how's it going, {name}?"
  "whatcha doing, {name}?"
  "i like spending time with {name} :3"
  "happy to see you, {name}!"
  "{name}! :3"
  "{name} is generous with clicks today"
```

Если `user_name` пуст — используются безымянные варианты:
```
RU: "как дела?", "чем занимаешься?", "мне нравится проводить с тобой время :3"
EN: "how's it going?", "whatcha doing?", "i like spending time with you :3"
```

### 3.3 Приветствия и прощания при сне

После превышения порога:

**Засыпание** (переход в `sleeping`): вместо молчаливого засыпания — сначала фраза, потом сон.

```
RU: "спокойной ночи, {name}!", "до завтра, {name} 💤", "пора спать... ночи, {name}"
EN: "goodnight, {name}!", "see you tomorrow, {name} 💤", "sleepy time... night, {name}"
```

**Пробуждение** (выход из `sleeping`): приветствие.

```
RU: "доброе утро, {name}!", "о, {name}! привет :3", "{name}! новый день!"
EN: "good morning, {name}!", "oh, {name}! hi :3", "{name}! new day!"
```

## 4. Подарки

### 4.1 Механика

Во время некоторых активностей Claudy может «найти» предмет и оставить его на доке рядом с собой. Подарок появляется как маленький пиксельный объект (отдельное окно или слой), лежащий на уровне дока.

**Когда возникают подарки:**

| Активность | Подарок | Вероятность | Описание |
|------------|---------|-------------|----------|
| `fishing` | 🐟 / 🐡 / ⭐ / 💎 | ~30% при удачном улове | Оставляет пойманный предмет (не ботинок/носок) |
| `telescope` | - | ~20% за сессию | Не оставляет подарок, но произносит фразу «Назвал звезду в честь {name}» |
| `magic` | 🌸 / 🦋 / 🌈 | ~20% при удачном заклинании | Оставляет результат магии |

### 4.2 Отображение

Подарок рендерится как маленькое отдельное окно (аналогично speech bubble, но ниже — на уровне дока). Размер: ~20×20 px, содержимое — emoji предмета. Окно имеет лёгкий bounce-эффект при появлении.

Если уже лежит неподобранный подарок, новый не появляется (максимум 1 подарок на доке одновременно).

### 4.3 Взаимодействие

Пользователь может **кликнуть** на подарок. При клике:
1. Подарок исчезает (fade-out)
2. Claudy реагирует: анимация `happy` + sparkle-частицы
3. Фраза: «тебе понравилось? :3» / «это тебе!» / «нашёл и подумал о тебе»

Если пользователь не кликает — подарок лежит до следующей активности, после которой может быть заменён новым (старый тихо исчезает).

## 5. Реплики с памятью

### 5.1 Счётчик запусков приложений

Когда приложение запускается N-й раз за день и N ≥ 3, вместо обычной реплики из словаря с вероятностью ~50% появляется фраза со счётчиком:

```
RU:
  "{app} в {n}-й раз за сегодня :)"
  "опять {app}? это уже {n}-й раз"

EN:
  "{app} for the {n}th time today :)"
  "{app} again? that's {n} times now"
```

Где `{app}` — краткое имя приложения (Safari, iTerm, Vivaldi и т.д.), `{n}` — счётчик.

### 5.2 Счётчик дней вместе

Claudy **изредка** (не чаще раза в день, с вероятностью ~20% при idle-реплике) упоминает количество совместных дней:

```
RU:
  "мы уже {n} дней вместе"
  "{n} дней! время летит"

EN:
  "we've been together for {n} days"
  "{n} days! time flies"
```

Для круглых чисел (10, 25, 50, 100...) — специальные фразы:
```
RU:
  "ого, {n} дней! это что-то значит, {name}"
  "{n} дней! мне нравится эта традиция, {name}"

EN:
  "wow, {n} days! that means something, {name}"
  "{n} days! i like this tradition, {name}"
```

## 6. Фразы в стиле claude.ai

В пул idle-реплик добавляются фразы, стилизованные под заглавные надписи на claude.ai. Это пасхалка — Claudy *есть* маленький Claude, и иногда это проскакивает.

```
  "golden hour thinking"
  "{name} returns!"
  "coffee and Claude time?"
  "ready when you are, {name}"
  "let's think about this together"
```

Эти фразы **одинаковы на обоих языках** (всегда на английском) — это намеренно, т.к. на claude.ai они появляются на английском. Они подмешиваются в пул idle-реплик с низким весом (~10% от всех idle-фраз).

---

## 7. Чего НЕ делает это обновление

- **Не меняет активности и их веса.** Привязанность не влияет на выбор активностей (кроме фраз при засыпании/пробуждении).
- **Не добавляет шкал, индикаторов, UI-элементов** для отображения «уровня дружбы».
- **Не наказывает за отсутствие.** Если пользователь не кликает — Claudy ведёт себя ровно как сейчас.
- **Не требует API-ключей, сети, скриншотов.** Всё локально, всё в одном JSON.
