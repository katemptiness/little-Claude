# Little Claude — Спецификация проекта

> Десктопный компаньон для macOS: автономный пиксельный крабик Claude, который живёт на рабочем столе, занимается своими делами и реагирует на пользователя и системные события.

---

## 1. Философия

Little Claude — **не тамагочи**. Он ничего не требует от пользователя. У него нет потребностей, индикаторов, здоровья, шкал голода. Он — самодостаточное маленькое существо со своей жизнью.

Пользователь может вообще не обращать на него внимания — он будет бродить по доку, читать книжку, ловить рыбу. Но если пользователь наведёт курсор или кликнет — крабик заметит и отреагирует.

Основные принципы:
- **Автономность**: крабик живёт своей жизнью, пользователь — наблюдатель и иногда собеседник
- **Ненавязчивость**: никаких принудительных уведомлений, попапов, блокировок
- **Характер**: крабик обладает «настроением» и «предпочтениями», зависящими от времени суток
- **Очарование**: каждая деталь должна вызывать улыбку

## 2. Визуальная реализация

### 2.1 Пиксельный формат

Крабик рендерится как спрайт на сетке **16×16 пикселей** с масштабированием ×5 (итого 80×80 точек). Каждый кадр — двумерный массив, где:
- `0` = прозрачный
- `1` = тело (терракотовый, `#D77757`)
- `2` = глаза (тёмный, `#2D2D2D`)
- `3` = румянец (`#F0C0A0`)
- `4` = реквизит: коричневый (книга, удочка) (`#7B5B3A`)
- `5` = реквизит: светлый (страницы, бумага) (`#F5F0E8`)
- `6` = реквизит: голубой (экран, вода) (`#60A5FA`)
- `7` = реквизит: фиолетовый (магия) (`#A855F7`)
- `8` = реквизит: серый (ноутбук) (`#8A8A9A`)
- `9` = реквизит: золотой (искры, звёзды) (`#FFD700`)

Спрайты рисуются программно через `CGContext` (как в референсном репозитории `pet-clawd`) и рендерятся один раз при инициализации. Переключение кадров — мгновенная замена `layer.contents`.

### 2.2 Анимации

Каждая активность — это **цикл из 2–5 спрайтовых кадров** с определённым интервалом и типом зацикливания. Анимации движения (bounce, shake, tilt, tremble) реализуются через перемещение `NSWindow` по экрану — без SpriteKit, без Metal, чистый AppKit.

Примитивы движения (реализованы в прототипах):
- **bounce**: серия вертикальных прыжков с затуханием
- **shake**: горизонтальные рывки с затуханием
- **tremble**: случайные смещения по обеим осям
- **tilt**: вращение спрайт-слоя через `CATransform3DMakeRotation`
- **squash**: сжатие/растяжение через `CATransform3DMakeScale`
- **pulse**: ресайз окна

### 2.3 Частицы

Эффекты (искры, сердечки, Z-ки, нотки, капли пота, восклицательные знаки) реализуются как:
- Либо пиксельные `CALayer`-оверлеи на спрайт-слое (как в `pet-clawd`)
- Либо отдельные мини-окна / слои с текстовыми символами (проще для сложных эффектов)

Каждая частица имеет: позицию, скорость, время жизни, затухание прозрачности.

## 3. Активности

### 3.1 Базовые состояния
| Состояние | Кадры | Описание |
|-----------|-------|----------|
| `idle` | 1 | Стоит, иногда моргает |
| `blink` | 1 | Морг (150мс, случайный интервал 2–6 сек) |
| `walk` | 2 | Бродит по доку, чередуя лапки |

### 3.2 Занятия
| Активность | Кадры | Частицы | Описание |
|------------|-------|---------|----------|
| `reading` | 3–4 | 📖 | Держит книгу в клешнях, глаза опущены, иногда «переворачивает страницу», иногда находит что-то интересное (глаза расширяются, `❗`) |
| `magic` | 4 | ✨🌈🦋 | Достаёт палочку → замахивается → ВЗМАХ → случайный результат (букет, радуга, звездопад, бабочка, «пуф») |
| `working` | 3 | `</>` `{ }` | Стучит по клавишам ноутбука, иногда задумывается, иногда яростно кодит |
| `fishing` | 4+ | 🐟👢💎 | Удочка по диагонали, ждёт, клюёт!, тянет, вытаскивает случайный предмет (рыба, ботинок, алмаз, носок…) с соответствующей реакцией |
| `sleeping` | 2 | 💤 | Тело «оседает», глаза закрыты, Z-ки уплывают вверх |
| `playing` | 2 | ♪♫ | Подпрыгивает, глаза прищурены, нотки |
| `music` | 2–3 | ♪♫♬ | Наушники (пиксельные), покачивается в такт |
| `painting` | 3–4 | 🎨 | Мольберт сбоку, мажет кистью, отходит, оценивает |
| `telescope` | 2 | ⭐ | Труба к глазу, пиксельные звёзды (преимущественно ночью) |
| `sandcastle` | 3–4 | — | Лепит, отходит, лепит ещё (ну, он же краб) |
| `meditating` | 1–2 | ✦ | Неподвижен, мерцающие частицы покоя, иногда парит на пиксель |
| `juggling` | 3 | 🔴🟢🔵 | Шарики по синусоиде |

### 3.3 Реактивные анимации (не самостоятельные, вызываются событиями)
| Реакция | Когда | Описание |
|---------|-------|----------|
| `happy` | клик, приятное событие | Глаза прищурены, румянец, bounce, искры |
| `love` | тройной клик | Глаза-сердечки, pulse, сердечки |
| `wave` | наведение курсора | Помахивание клешнёй |
| `look_up` | приложение запущено | Задирает голову вверх, бежит к иконке |
| `yawn` | пробуждение Mac | Потягивается, зевает |
| `surprise` | неожиданное событие | Широкие глаза, jump, `❗` |

### 3.4 Переходные анимации
Между состояниями — мягкие переходы: потягивание после сна, зевок перед засыпанием, закрытие книги перед ходьбой.

## 4. Стейт-машина поведения

### 4.1 Автономные переходы

Крабик переходит между активностями по взвешенным вероятностям. Не существует фиксированной последовательности — всё стохастически, что создаёт ощущение «собственной воли».

```
[текущая активность завершена]
    → выбор следующей по весам, зависящим от времени суток
    → переходная анимация
    → новая активность
```

Каждая активность имеет:
- **минимальную длительность** (не прервётся раньше)
- **максимальную длительность** (после неё — переход)
- **вес по времени суток** (см. 4.2)

### 4.2 Режим дня (Night Owl)

Пользователь — сова. Режим по умолчанию:

| Период | Часы | Поведение |
|--------|------|-----------|
| Глубокий сон | 04:00 – 11:00 | Спит. Будить можно кликом, но быстро засыпает снова. |
| Утро (для совы) | 11:00 – 13:00 | Просыпается, потягивается, зевает. Вялые активности: бродить, медитировать. |
| День | 13:00 – 20:00 | Полный набор активностей, активные преобладают. |
| Вечер | 20:00 – 01:00 | Спокойные активности: чтение, телескоп, рыбалка, музыка. |
| Поздняя ночь | 01:00 – 04:00 | Всё тише: дремлет, читает, смотрит в телескоп. Засыпает. |

> **Будущее:** окно настроек с ползунками для настройки режима дня.

### 4.3 Прерывание активностей

Системные события и взаимодействие с пользователем могут прервать текущую активность:
- Клик → прерывание → реакция → возврат к делам
- Системное событие (запуск приложения) → прерывание → реакция → возврат
- Сон Mac → прерывание → засыпает

Приоритет прерываний: системный сон > клик пользователя > таймер переключения.

## 5. Взаимодействие с пользователем

### 5.1 Курсор

| Событие | Реакция |
|---------|---------|
| Курсор приблизился к крабику | Поворачивается к курсору, может замереть |
| Курсор наведён на крабика | Машет клешнёй, может сказать фразу |
| Курсор ушёл | Возвращается к своим делам |

### 5.2 Клики

| Событие | Реакция |
|---------|---------|
| Одиночный клик | Варианты: радуется (bounce, sparkle), может сказать фразу, может «влюбиться» (влюбленные глаза, сердечки) |
| Двойной клик | Открывает приложение Claude.app |
| Перетаскивание | Удивляется, при отпускании — падает с гравитацией и отскоком (как в `pet-clawd`) |

### 5.3 Контекстное меню (правый клик)
- Настройки (будущее: окно настроек)
- Открыть Claude
- О программе
- Выход

## 6. Реакции на системные события

### 6.1 Запуск приложений (NSWorkspace.didLaunchApplicationNotification)

Крабик получает bundle ID запущенного приложения и может:
1. Подбежать к области дока
2. Задрать голову (анимация `look_up`)
3. Сказать фразу из словаря

Примерный словарь реплик:

```
Safari / Vivaldi / Chrome   → "о, опять сидим в Интернете?"
Terminal / iTerm            → "хакерское время!"
Sublime Text / VS Code      → "кодим? кодим."
Spotify / Music             → "о, музыка! 🎵"
Telegram                    → "кто-то написал?"
Finder                      → "ищем что-то?"
Photos                      → "о, фоточки!"
```

Реплики хранятся как словарь `[bundleID: [String]]` — легко расширяется.

### 6.2 Системные события

| Событие | API | Реакция |
|---------|-----|---------|
| Mac засыпает | `NSWorkspace.willSleepNotification` | Крабик ложится спать (анимация, Z-ки) |
| Mac просыпается | `NSWorkspace.didWakeNotification` | Потягивается, зевает |
| Переключение приложения | `didActivateApplicationNotification` | Может повернуться, может проигнорировать |
| Подключение наушников | `AVAudioSession` / `CoreAudio` | "о, будем слушать музыку?" |
| Смена тёмной/светлой темы | `NSApp.effectiveAppearance` | Может сказать что-то, может не заметить |
| Подключение к WiFi | `NWPathMonitor` | "мы в сети!" / "ой, WiFi пропал..." |

## 7. Речевые пузыри

### 7.1 Формат

Маленькое полупрозрачное окно над крабиком с текстом. Появляется с fade-in, висит 3–5 секунд, исчезает с fade-out. Визуально — закруглённый прямоугольник, шрифт моноширинный, стиль pet-clawd.

### 7.2 Категории фраз

**Привязанные к активностям:**
```
reading:    "хмм, интересно...", "ещё одну главу...", "а, вот оно что!"
fishing:    "тихо... не спугни...", "кажется, клюёт!", "...это же носок"
working:    "тук-тук-тук", "баг или фича?", "git push --force"
magic:      "абракадабра!", "хмм, не то...", "✨"
sleeping:   (без фраз, только Z-ки)
meditating: "ом...", "🧘", "..."
telescope:  "о, вон та звезда...", "космос...", "красиво..."
```

**Случайные (idle):**
```
":3", "привет!", "хороший день", "...", "*щёлк-щёлк*"
```

**Реакции на пользователя:**
```
клик:       "привет! :3", "о!", "рада тебя видеть"
наведение:  "хм?", "*машет*"
```

**Реакции на события:** — см. раздел 6.

### 7.3 Частота

Фразы появляются **редко** — не чаще одного раза в 30–60 секунд. Цель — приятная неожиданность, не спам. Фразы при активностях показываются не каждый цикл, а с вероятностью ~30%.

## 8. Техническая архитектура

### 8.1 Платформа

- **macOS** (Apple Silicon)
- **Python + PyObjC** (предпочтительно, знакомый стек пользователя)
- Альтернатива: Swift / AppKit (более нативно, но менее знакомо)

### 8.2 Компоненты

```
little-claude/
├── app.py                  # Точка входа, NSApplication
├── character.py            # Стейт-машина, логика поведения
├── sprite_renderer.py      # Рендеринг спрайтов (CGContext → CGImage)
├── sprites/
│   ├── base.py             # idle, blink, walk
│   ├── activities.py       # reading, fishing, magic, working, ...
│   └── reactions.py        # happy, love, surprise, wave, ...
├── animations.py           # Примитивы: bounce, shake, tilt, ...
├── particles.py            # Система частиц
├── speech.py               # Речевые пузыри
├── system_events.py        # NSWorkspace подписки
├── schedule.py             # Режим дня, веса активностей
├── config.py               # Конфигурация, пути, константы
└── resources/
    └── phrases.json        # Словарь фраз
```

### 8.3 Ключевые технические решения

**Окно:**
- `NSWindow` с `styleMask: .borderless`, прозрачный фон
- Уровень `CGWindowLevelForKey(.maximumWindow)` — поверх всего
- `collectionBehavior: [.canJoinAllSpaces, .fullScreenAuxiliary, .stationary]`
- `ignoresMouseEvents = false` для перехвата кликов

**Основной цикл:**
- `CVDisplayLink` или `NSTimer` (60 fps) для `update()`
- `update()` обновляет: стейт-машину, позицию окна, анимацию, частицы, моргание

**Спрайты:**
- Массивы `16×16` → `CGImage` через `CGContext` при инициализации
- `CALayer.contents` для мгновенного переключения кадров
- `CATransaction.setDisableActions(true)` для отключения имплицитных анимаций

**Позиционирование на доке:**
- Крабик ходит по нижней кромке экрана (или по верхнему краю дока)
- `NSScreen.main.frame` для границ
- Dock position/size через `NSScreen.visibleFrame` (разница с `frame` = размер дока)

### 8.4 Зависимости

- Python 3.x
- PyObjC (`pip install pyobjc`)
- Опционально: `py2app` для сборки в .app

## 9. Будущие идеи

- **Окно настроек**: режим дня (ползунки), частота фраз, набор активностей (вкл/выкл), громкость (если добавить звуки)
- **Звуки**: тихие пиксельные звуки при активностях (опционально, выкл по умолчанию)
- **Сезонные скины**: шапка зимой, цветок весной, солнечные очки летом
- **Друзья**: второй крабик, который иногда появляется и взаимодействует с первым
- **Достижения**: скрытые достижения за редкие события (поймал алмаз 3 раза подряд)

## 10. Референсные материалы

В папке проекта находятся:

- `reference/clawd-tamagotchi.jsx` — первый прототип (тамагочи-стиль с потребностями, частицами, стейт-машиной). Полезен как референс для: базовых спрайтов, системы частиц, game loop, рендеринга через canvas.

- `reference/clawd-activities.jsx` — демо четырёх активностей (чтение, магия, ноутбук, рыбалка). Полезен как референс для: фазовых анимаций, реквизита в спрайтах, системы случайных результатов (рыбалка), привязки частиц к фазам.

- Репозиторий `https://github.com/getcompanion-ai/pet-clawd` (MIT) — нативная macOS-реализация похожей концепции на Swift/AppKit. Ключевые файлы для изучения:
  - `CrabCharacter.swift` — стейт-машина, анимации, перетаскивание, гравитация
  - `CrabSpriteRenderer.swift` — пиксельный рендеринг через CGContext
  - `CrabContentView.swift` — обработка кликов и drag
  - `ClawdApp.swift` — настройка NSApplication

Все решения из `pet-clawd` переносятся в PyObjC один-в-один.
